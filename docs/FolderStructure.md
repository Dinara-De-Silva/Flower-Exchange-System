## Folder Descriptions

### `src/`
Contains all C++ source (`.cpp`) implementation files, organized into subdirectories by responsibility.

| File | Description |
|------|-------------|
| `main.cpp` | Entry point of the application. Parses the run command and hands control over to `Exchange`. No business logic lives here. |
| `Exchange.cpp` | **Core logic starts here.** Orchestrates the full exchange workflow — reads input orders, dispatches them to the appropriate `Orderbook`, and triggers execution report generation. |
| `src/core/Flower.cpp` | Implements the `Flower` type/enum representing the five traded instruments: Rose, Lavender, Lotus, Tulip, and Orchid. Includes any instrument-level utilities. |
| `src/core/Order.cpp` | Implements the `Order` class. Represents a single client order with fields such as Client Order ID, Client, Instrument, Side, Quantity, and Price. |
| `src/core/Orderbook.cpp` | Implements the `Orderbook` class for a single flower instrument. Manages the buy and sell sides and drives the order-matching logic. |
| `src/core/OrderbookSide.cpp` | Implements `OrderbookSide`, which maintains an ordered collection of resting orders for one side (buy or sell) of an `Orderbook`. |
| `src/io/CSVReader.cpp` | Implements `CSVReader`. Reads and parses the input `orders.csv` file into `Order` objects for processing. |
| `src/io/CSVWriter.cpp` | Implements `CSVWriter`. Writes execution report entries to the output `execution_rep.csv` file. |

### `src/api/`
Reserved for future API-layer source files (e.g., external connectivity, FIX protocol adapters, or REST endpoints).

---

### `include/`
Contains all C++ header (`.h`) files, mirroring the `src/` subdirectory layout.

| File | Description |
|------|-------------|
| `include/Exchange.h` | Declares the `Exchange` class and its public interface. Entry point for all exchange-level orchestration. |
| `include/core/Flower.h` | Declares the `Flower` type representing the five traded instruments. |
| `include/core/Order.h` | Declares the `Order` class and its attributes. |
| `include/core/Orderbook.h` | Declares the `Orderbook` class managing a single instrument's order book. |
| `include/core/OrderbookSide.h` | Declares `OrderbookSide`, managing one side (buy/sell) of an order book. |
| `include/io/CSVReader.h` | Declares `CSVReader` for reading and parsing the input CSV file. |
| `include/io/CSVWriter.h` | Declares `CSVWriter` for writing the output execution report CSV file. |

### `include/api/`
Reserved for future API-layer header files.

---

### `data/`
Holds all CSV data files used by the system.

#### `data/input/`
- `orders.csv` — Input file containing client orders. Fields: `Client Order ID`, `Client`, `Instrument`, `Side`, `Quantity`, `Price`.

#### `data/output/`
- `execution_rep.csv` — Output file generated by the system after processing all orders. Contains execution report entries with statuses: `New`, `Fill`, `PFill`, `Reject`.

---

### `tests/`
Contains files used for testing and validating system output.

#### `tests/input/`
Test-specific input CSV files used to drive individual test scenarios.

#### `tests/expected_output/`
Expected execution report CSV files corresponding to each test input, used to validate correctness of system output.

---

### `docs/`
Contains project documentation.

- `FolderStructure.md` — This file. Documents the full folder layout and the purpose of every component.

---

## Entry Point & Execution Flow

1. **`main.cpp`** — Receives the run command. Instantiates and starts the `Exchange`.
2. **`Exchange.cpp`** — All exchange initialization and processing begins here. Uses `CSVReader` to load orders, routes each order to the matching `Orderbook`, and uses `CSVWriter` to produce the execution report.

### Compile & Run

```bash
# Compile
g++ -std=c++17 -Iinclude -o Exchange \
    src/main.cpp \
    src/Exchange.cpp \
    src/core/Flower.cpp \
    src/core/Order.cpp \
    src/core/Orderbook.cpp \
    src/core/OrderbookSide.cpp \
    src/io/CSVReader.cpp \
    src/io/CSVWriter.cpp

# Run
./Exchange
```

> The executable is named `Exchange` reflecting that `Exchange.cpp` is the heart of the system.
